# 04 – Policies and Controls

This module provides a set of sample policies, standards, and control mappings tailored to healthcare environments. These documents reflect governance best practices and align with regulatory and operational realities.

## Included Policies

### 1. Access Control Policy
Defines role-based access, least privilege, authentication requirements, and periodic access reviews.

### 2. Data Protection Policy
Covers PHI handling, encryption requirements, data retention, and secure data transfer.

### 3. Cloud Governance Policy
Outlines cloud resource provisioning, tagging standards, monitoring expectations, and compliance guardrails.

### 4. AI Usage Policy (Healthcare-Specific)
Defines acceptable use of AI tools, PHI restrictions, model risk considerations, and human oversight requirements.

### 5. Incident Response Policy
Establishes roles, escalation paths, communication protocols, and post-incident review expectations.

## Control Mapping

Controls will be mapped to:
- NIST 800-53
- NIST CSF
- HIPAA Security Rule
- PHIPA (where applicable)
- ISO 27001 (select controls)

## Objective

Provide healthcare organizations with a practical, adaptable set of policies and controls that strengthen governance, reduce risk, and support compliance.
